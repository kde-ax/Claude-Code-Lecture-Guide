# 아웃풋/ — 위키에서 파생된 산출물

> 위키(`wiki/`)는 지식 그 자체, 이 폴더는 **그 지식으로 만든 결과물**이다.
> 글로벌 `~/.claude/CLAUDE.md`의 3가지 스킬(사용 안내서·히스토리·슬랙 공지)이 여기로 떨어진다.

---

## 하위 구조

```
아웃풋/
├── 사용안내서/    ← 960×540 HTML 슬라이드 (PDF 변환용)
├── 히스토리문서/  ← .docx 프로젝트 구축 가이드
└── 슬랙공지/      ← .md 또는 .txt 슬랙 메시지
```

---

## 파일명 규칙

| 폴더 | 패턴 | 예시 |
|------|------|------|
| `사용안내서/` | `[기능명]_사용안내서_YYYYMMDD.html` | `회의실예약_사용안내서_20260504.html` |
| `히스토리문서/` | `[프로젝트명]_구축가이드_히스토리_YYYYMMDD.docx` | `회의실예약_워크플로우_구축가이드_히스토리_20260504.docx` |
| `슬랙공지/` | `[기능명]_슬랙공지_YYYYMMDD.md` | `회의실예약_슬랙공지_20260504.md` |

## 산출물 작성 원칙

### 공통
- 한국어
- CCFM 메인 블루: `#1B4FD8`
- 라이트 블루 배경: `#EFF3FF`
- 산출물의 **각 주장은 위키 페이지에서 인용** — 가능하면 frontmatter나 푸터에 출처 위키 페이지 명시

### 사용 안내서 슬라이드
- 글로벌 `CLAUDE.md` §스킬1의 모든 규격 준수 (960×540, `overflow: hidden`, Pretendard, `@page` 인쇄설정 등)
- 슬라이드 구성: 표지 → 왜 만들었나 → 무엇을 할 수 있나 → 기능별 상세 → 결과 화면 → 주의/팁

### 히스토리 문서 (.docx)
- 글로벌 `CLAUDE.md` §스킬2 구조 준수 (배경 / 완성된 기능 / 구축 과정 step / 트러블슈팅 / UX 고민 / 남은 작업)
- **재현·인수인계용**이지 보고용이 아님 — 담당자가 바뀌어도 처음부터 다시 만들 수 있어야 함

### 슬랙 공지
- 글로벌 `CLAUDE.md` §스킬3 템플릿 준수 (해요체, 이모지 매핑)
- 배경 1~2문장, 기능 3~5개, DM 안내 마무리

## 위키 → 아웃풋 흐름

```
raw/ (원본)
  ↓ ingest
wiki/sources/, wiki/entities/, wiki/concepts/
  ↓ 사용자 요청 ("회의실 예약 사용 안내서 만들어줘")
아웃풋/사용안내서/회의실예약_사용안내서_20260504.html
```

산출물은 위키를 직접 참조해 만든다. 위키에 정보가 없으면 먼저 ingest 또는 보강 후 만들 것.

---

## 산출물 작성 후

1. 파일을 `아웃풋/` 적절한 폴더에 저장
2. `wiki/log.md`에 한 줄 추가:
   ```
   ## [2026-05-04] output | 회의실예약 사용 안내서
   - 산출: 아웃풋/사용안내서/회의실예약_사용안내서_20260504.html
   - 참조: [[n8n]], [[회의실예약-워크플로우]]
   ```
3. 산출 결과를 위키 페이지로도 남길 가치가 있으면 (예: 새로운 결정·합성) `wiki/synthesis/`에 페이지 생성 제안
