## 정의

RNN(순환 신경망)은 **시퀀스 데이터를 한 단계씩 순서대로 처리하면서 이전 상태를 다음 단계로 넘기는** [[신경망]] 아키텍처다 [[20260504_AWS-LLM이란]]. [[트랜스포머]]가 등장하기 전, 자연어 처리·번역·시계열 예측의 표준이었다.

"recurrent(순환)"라는 이름은, 같은 신경망 블록이 시퀀스의 각 단계에서 반복적으로 호출된다는 뜻에서 왔다.

## 핵심 메커니즘

```
입력: ["나", "는", "학생", "입니다"]

1단계: "나" 입력 → 은닉상태 h1 출력
2단계: "는" + h1 입력 → h2 출력
3단계: "학생" + h2 입력 → h3 출력
4단계: "입니다" + h3 입력 → h4 출력
```

각 단계가 이전 단계의 결과(은닉 상태, hidden state)를 함께 받기 때문에, **이전 정보가 누적**된다.

## 일상 비유

릴레이 마라톤이라고 보면 된다.
- 1번 주자가 뛰고 → 바통(은닉 상태)을 2번에게 넘김
- 2번이 1번의 바통을 받아 뛰고 → 3번에게 넘김
- ...
- 마지막 주자는 모든 이전 주자의 영향을 누적해 받음

장점: 이전 정보가 자연스럽게 흐름으로 전달됨.
단점: **앞 주자가 멀어질수록 바통의 정보가 흐릿해진다** (장기 의존성 문제).

## 한계 — 왜 [[트랜스포머]]에 밀렸나

### 1. 순차 처리 → 느리다
시퀀스를 단계적으로 처리해야 하므로 **GPU의 병렬 연산을 활용 못 함**. 긴 문장은 처리 시간이 길이에 비례해 늘어남.

### 2. 장기 의존성 약함
"한국에서 태어났고… (50단어 후) … 그래서 그는 ___를 잘한다"
→ 빈칸에 "한국어"가 와야 하지만, RNN은 50단어 전 정보가 흐려져서 못 잡는 경우가 많다.

LSTM, GRU 같은 RNN 개량형이 이 문제를 어느 정도 해결했지만 근본적인 한계는 남았다.

### 3. [[트랜스포머]]의 등장
2017년 [[트랜스포머]]가 등장하며:
- 시퀀스를 병렬로 처리 (속도↑)
- [[셀프어텐션]]으로 거리에 무관하게 토큰 관계 잡음 (장기 의존성 해결)

→ 자연어 처리에서 RNN은 빠르게 트랜스포머에 자리를 내줬다.

## 아직 RNN이 쓰이는 곳

- **시계열 예측** (주가, 센서 데이터): 트랜스포머도 진출 중이지만 여전히 RNN 변형이 경쟁력
- **저자원 환경**: 트랜스포머보다 메모리·연산 부담이 가벼움
- **온라인 학습**: 데이터가 들어오는 대로 점진적으로 학습할 때
