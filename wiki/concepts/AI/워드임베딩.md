## 정의

워드 임베딩은 **단어를 다차원 숫자 벡터로 변환하는 방법**이다. 의미가 비슷한 단어가 벡터 공간에서 서로 가깝게, 다른 단어는 멀게 배치되도록 학습된다 [[20260504_AWS-LLM이란]].

이 변환이 있어야 [[신경망]]이 단어를 "이해"할 수 있다 — 컴퓨터는 글자가 아니라 숫자만 다룰 줄 알기 때문.

## 핵심 메커니즘

옛날 방식(원-핫 인코딩):
- 단어를 매우 긴 0/1 벡터로 표현 ([0,0,0,...,1,...,0,0])
- 단어 간 의미 관계가 전혀 없음 (모든 단어 쌍이 같은 거리)

워드 임베딩:
- 단어를 짧고 빽빽한 실수 벡터 (예: 768차원의 [0.12, -0.45, 0.88, ...])
- **의미가 비슷하면 가까이, 다르면 멀리** 위치하도록 학습됨

학습은 "비슷한 문맥에서 등장하는 단어는 의미가 비슷하다"는 분포 가설(distributional hypothesis)에 기반한다. Word2Vec, GloVe, FastText 같은 초기 방법이 있었고, [[트랜스포머]] 시대에는 학습된 임베딩이 모델 안에 내장된다.

## 일상 비유

도시 지도라고 생각하면 된다.
- "강남"과 "서초" → 가까운 좌표 (둘 다 서울 강남권)
- "강남"과 "부산" → 먼 좌표 (다른 도시)

워드 임베딩은 단어 지도다. 의미적으로 가까운 단어들이 지도상에서도 가까이 모인다.

심지어 **벡터 연산**으로 의미 추론도 된다:
```
king - man + woman ≈ queen
서울 - 한국 + 일본 ≈ 도쿄
```

벡터의 방향 자체가 의미를 인코딩한다는 뜻이다.

## [[LLM]]에서의 역할

[[LLM]]에서 입력 텍스트는 다음 흐름을 탄다:

```
"안녕하세요" 
  → [[토큰]]화: ["안녕", "하세요"]
  → 워드 임베딩: 각 토큰이 768차원(또는 더 큰) 벡터로 변환
  → [[트랜스포머]]에 입력
  → [[셀프어텐션]]으로 문맥 반영
  → 다음 토큰 예측
```

워드 임베딩이 없으면 트랜스포머가 단어를 처리할 입력이 아예 없다. 즉, [[LLM]]의 입구다.

## 관련 개념

- 상위: [[신경망]], [[딥러닝]]
- 함께: [[토큰]] (임베딩의 단위), [[트랜스포머]] (임베딩을 받는 모델)
- 비교: 원-핫 인코딩 (이전 세대)
- 응용: [[LLM]], 검색, 추천 시스템
