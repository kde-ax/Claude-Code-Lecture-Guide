## 정의

토큰은 [[LLM]]이 텍스트를 다룰 때의 **기본 단위**다. 한 토큰은 단어 하나일 수도 있고, 글자 몇 개의 조각일 수도 있다 [[20260504_AWS-LLM이란]].

LLM은 글자도 단어도 아닌 **토큰** 단위로 입력을 받고 출력을 만든다. 이 변환을 해주는 도구를 **토크나이저(tokenizer)**라고 한다.

## 핵심 메커니즘

토크나이저는 텍스트를 잘게 쪼갠다. 영어는 보통 다음과 같이:
- "ChatGPT is amazing" → ["Chat", "G", "PT", " is", " amazing"]

영어 단어 하나가 1~4토큰 정도, 한국어는 자모 단위로 더 잘게 쪼개져서 일반적으로 영어보다 토큰이 더 많이 든다.

대표적 토크나이저 알고리즘:
- **BPE (Byte-Pair Encoding)**: GPT 계열에서 사용
- **WordPiece**: BERT에서 사용
- **SentencePiece**: 다국어 모델에서 사용

## 일상 비유

레고 블록이라고 생각하면 된다.
- 글자 = 레고 부품의 가장 작은 알갱이
- **토큰 = 자주 같이 쓰이는 부품 묶음** (예: "ing", "tion", "Chat" 등)
- 단어 = 토큰 여러 개 조합
- 문장 = 단어 여러 개 조합

LLM은 글자 알갱이를 매번 다시 조립하는 게 아니라, 자주 쓰는 묶음(토큰) 단위로 처리해서 효율을 높인다.

## 왜 토큰을 알아야 하나

LLM의 모든 비용·제약이 토큰 단위로 매겨진다.

### 1. API 가격
[[OpenAI]], [[Claude]] 등 [[LLM]] 서비스는 **토큰당 과금**한다. "1,000 입력 토큰당 $X, 1,000 출력 토큰당 $Y" 식.

### 2. 컨텍스트 윈도우
LLM이 한 번에 처리할 수 있는 토큰 수에 한계가 있다.
- [[GPT-3]] 초기: 약 4,000 토큰
- [[Claude]] 2: 100,000 토큰 (책 한 권)
- 최신 모델들: 1,000,000 토큰까지

### 3. 한글은 토큰 효율이 나쁘다
한글은 자모 단위로 쪼개지기 쉬워 같은 의미를 영어보다 1.5~2배 많은 토큰으로 표현하게 된다 → 같은 분량이면 한글이 더 비쌈.

## 실전 팁

- 프롬프트가 길어지면 비용·지연이 커진다 → 핵심만 추려 보내기
- 한글로 쓰면 토큰이 더 들어가니, 영어 키워드를 섞으면 효율적일 때가 있다
- OpenAI의 [tiktoken](https://github.com/openai/tiktoken) 같은 라이브러리로 사전에 토큰 수를 셀 수 있다
