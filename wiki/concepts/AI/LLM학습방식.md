## 정의

[[LLM]]을 특정 작업에 적용하는 방식은 크게 **3종**으로 나뉜다 [[20260504_AWS-LLM이란]]:

1. **제로샷 학습 (Zero-shot)**: 별도 예시 없이 지시만 주고 바로 답하게 한다
2. **퓨샷 학습 (Few-shot)**: 예시 몇 개를 함께 주어 패턴을 잡게 한다
3. **미세조정 (Fine-tuning)**: 추가 데이터로 [[파라미터]] 자체를 다시 조정한다

위 3개는 "예시를 얼마나, 어떤 방식으로 주느냐"의 차이다. 위로 갈수록 가볍고, 아래로 갈수록 무겁다 (비용·시간·데이터).

## 1. 제로샷 학습

```
프롬프트: "다음 문장을 영어로 번역해: 안녕하세요"
LLM 응답: "Hello"
```

별도 학습 없이 [[LLM]]이 사전학습으로 이미 알고 있는 지식만으로 답한다. 가장 빠르고 가장 저렴하지만, 정확도는 케이스마다 들쑥날쑥.

## 2. 퓨샷 학습

```
프롬프트:
사과 → 빨강
바나나 → 노랑
포도 → ?

LLM 응답: 보라
```

질문 앞에 **예시 몇 개**(few = 2~10개)를 보여주면 LLM이 "아, 이런 형식으로 답하면 되는구나" 하고 패턴을 잡는다. 제로샷보다 정확도가 크게 올라가는 경우가 많다.

이건 [[파라미터]]를 건드리지 않고 단지 프롬프트 안에서 예시를 보여주는 방식이라 **인-컨텍스트 학습 (In-context Learning)**이라고도 부른다. 모델은 변하지 않는다.

## 3. 미세조정 (Fine-tuning)

```
1. 베이스 모델 ([[GPT-3]] 등)을 가져온다
2. 우리 도메인 데이터 (예: 법률 문서 1만 건) 준비
3. 베이스 모델을 그 데이터로 추가 학습 → [[파라미터]] 값이 미세하게 바뀜
4. "법률 전문 GPT" 모델 완성
```

추가 학습으로 모델 자체를 우리 용도에 맞춘다. 정확도는 가장 높지만:
- 학습 데이터 (수백~수만 건) 필요
- GPU 비용
- 모델 호스팅 부담

## 일상 비유

새 직원에게 일을 시키는 3가지 방식이다.

| 방식 | 비유 |
|---|---|
| **제로샷** | "이거 처리해줘" — 직원이 알아서 함. 빠르지만 결과가 들쭉날쭉 |
| **퓨샷** | "이거 처리해줘. 예전에 이런 식으로 처리한 사례가 있어, 봐봐" + 샘플 3개 — 직원이 패턴 잡고 처리 |
| **미세조정** | 직원을 우리 회사 도메인 학원에 1주일 보내 추가 교육시킨 다음 일을 시킴 — 정확하지만 시간·돈 듬 |

## 어떤 걸 언제 쓸까

| 상황 | 추천 |
|---|---|
| 일반적인 질문, 빠른 프로토타입 | 제로샷 |
| 출력 형식이 일정해야 함, 정확도 살짝 더 필요 | 퓨샷 (예시 3~5개) |
| 특정 도메인(법률·의료·내부 데이터)에 특화 필요 | 미세조정 |
| 고정된 작업 + 대용량 호출 | 미세조정 (장기적으로 비용 우위) |

요즘은 **RAG (Retrieval-Augmented Generation)**이라는 또 다른 방식도 인기. 외부 지식을 검색해서 프롬프트에 넣어주는 방식인데, 미세조정 없이도 도메인 특화가 가능해 자주 채택된다.
