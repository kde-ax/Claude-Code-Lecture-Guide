## 정의

파라미터는 **[[신경망]] 안의 가중치(weight)와 편향(bias) 전부**, 그리고 [[워드임베딩]]을 묶어 부르는 말이다. 모델이 학습 과정에서 조정하는 숫자들이다 [[20260504_AWS-LLM이란]].

"이 모델은 1,750억 파라미터"라는 말은 **신경망 안에 175,000,000,000개의 숫자가 들어있고, 학습 중에 그 숫자들을 미세하게 조정해 정답에 가까워지게 만든다**는 뜻이다.

## 핵심 메커니즘

훈련은 본질적으로 **파라미터 조정 게임**이다:
1. 입력에 대해 모델이 출력을 만든다
2. 정답과 비교해 오차(loss)를 측정한다
3. 오차를 줄이는 방향으로 모든 파라미터를 조금씩 움직인다 (역전파, gradient descent)
4. 1~3을 수십억 번 반복

훈련이 끝나면 파라미터 값들이 고정된다. 추론(inference) 시에는 이 고정된 값으로 출력만 계산한다.

## 일상 비유

라디오의 다이얼이라고 생각하면 된다.
- 라디오 하나에 다이얼이 1,750억 개 달려 있다
- 처음에는 다이얼이 다 무작위 위치
- "이 입력에는 이런 답이 정답이야"라고 가르칠 때마다 다이얼들을 아주 살짝 조정
- 수십억 번 가르치면 다이얼들이 점점 정답을 잘 내는 위치에 자리잡는다

훈련이 끝난 LLM = 다이얼이 모두 "정답을 내는 자리"에 고정된 거대한 라디오.

## 모델 크기와 파라미터

| 모델 | 파라미터 수 |
|---|---|
| BERT-base | 1.1억 |
| GPT-2 | 1.5억 |
| [[GPT-3]] | 1,750억 |
| Jurassic-1 | 1,780억 |
| GPT-4 | 비공개 (수조 추정) |
| [[Llama-2]] 70B | 700억 |

일반적으로:
- 파라미터가 많을수록 → 더 많은 패턴을 외울 수 있음
- 단, 그만큼 훈련 비용·추론 비용·메모리 사용도 폭증
- "파라미터 수 vs. 성능"은 단순 비례가 아님 — 데이터 품질, 아키텍처, 학습 방법이 다 영향

## Chinchilla 법칙 (참고)

DeepMind의 2022년 연구에 따르면, **모델 크기와 학습 데이터 크기의 균형**이 중요하다. 파라미터만 키우고 데이터가 부족하면 성능이 안 올라온다. 그래서 최근에는 무작정 크게 만들기보다 데이터 품질·양에도 똑같이 투자한다.
