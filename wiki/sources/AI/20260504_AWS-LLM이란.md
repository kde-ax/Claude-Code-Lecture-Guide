## 한 줄 요약

LLM이 무엇인지(트랜스포머 기반의 초대형 딥러닝 모델), 어떻게 작동하는지(워드 임베딩 + 셀프어텐션 + 다음 토큰 예측), 어디에 쓰는지(카피라이팅·코드 생성·지식 답변 등)를 입문자 눈높이로 설명하는 AWS 공식 문서.

## 핵심 takeaway

- **LLM = 트랜스포머 기반 초대형 딥러닝 모델.** 사전 학습 단계에서 인터넷 규모(Common Crawl, Wikipedia 등) 데이터를 자체 학습으로 흡수한다.
- **트랜스포머의 결정적 차이**: RNN과 달리 시퀀스를 **병렬 처리**해 GPU에서 빠르게 훈련된다 → 수천억 개 파라미터 모델이 가능해짐.
- **워드 임베딩**으로 단어를 다차원 벡터로 표현해 "의미가 비슷한 단어가 벡터 공간에서 가까이" 위치하도록 만든다. 이게 LLM이 문맥을 이해하는 기반.
- **학습 방식 3종**: 제로샷(별도 훈련 없이 바로 답) / 퓨샷(예시 몇 개로 성능 향상) / 미세조정(추가 데이터로 파라미터 조정).
- **대표 모델**: GPT-3(1,750억 파라미터), Claude 2(10만 토큰 컨텍스트), Llama 2, AI21 Jurassic-1, Cohere Command.
- **응용**: 카피라이팅, 지식 답변(KI-NLP), 텍스트 분류, 코드 생성(Copilot/CodeWhisperer/Codex), 텍스트 생성.
- **AWS 측 서비스**: Amazon Bedrock(여러 LLM을 API로 호출), SageMaker JumpStart(파운데이션 모델 허브).

## 주요 인용

> "대규모 언어 모델(LLM)은 방대한 양의 데이터로 사전 학습된 초대형 딥 러닝 모델입니다. 기본 트랜스포머는 셀프 어텐션(self-attention) 기능을 갖춘 인코더와 디코더로 구성된 신경망 세트입니다."

> "입력을 순차적으로 처리하는 이전의 순환 신경망(RNN)과 달리 트랜스포머 전체 시퀀스를 병렬로 처리합니다. 이를 통해 데이터 사이언티스트는 GPU를 사용하여 트랜스포머 기반 LLM을 학습할 수 있어 훈련 시간을 크게 줄일 수 있습니다."

> "다차원 벡터를 사용하여 벡터 공간에서 문맥상 의미가 비슷하거나 다른 관계가 있는 단어가 서로 가깝도록 단어를 표현함으로써 이러한 한계를 극복했습니다."

## 한계 / 비판 지점

- AWS의 마케팅 컬러가 들어간 입문 자료 — 본문 후반부는 Bedrock·SageMaker 영업으로 넘어간다. LLM 일반론으로만 인용할 부분과 AWS 서비스 소개 부분을 구분해서 읽어야.
- 트랜스포머의 작동 원리(어텐션 수식, 학습 손실 함수 등)는 거의 설명 없음 — 입문자용으로는 충분하지만 실제 구현 이해에는 부족.
- "비지도 학습"과 "자체 학습(self-supervised)"을 구분하면서도 다소 모호하게 설명. 정확히는 자기 지도 학습(self-supervised learning)이 더 표준 용어.
