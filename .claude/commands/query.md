# /query — 위키에 질문하기

CLAUDE.md `§3.2 Query` 워크플로우를 그대로 수행한다. **위키 안에 있는 사실로만 답한다.** 외부 검색은 사용자가 명시 요청한 경우에만.

## 입력

- 질문: `$ARGUMENTS`
- 비어 있으면 사용자에게 무엇을 묻고 싶은지 한 문장으로 받기.

## 절차

### 1. 인덱스 우선

- `wiki/index.md`를 먼저 Read.
- 질문에 관련된 area/type을 추리고, **후보 페이지 목록**을 사용자에게 짧게 노출 (3~7개).
- 인덱스가 비어 있거나 너무 얕으면 `wiki/{type}/{area}/`를 Glob으로 직접 훑는다.

### 2. 드릴다운

- 후보 페이지들을 Read해서 본문과 frontmatter `sources`를 함께 본다.
- 위키링크가 가리키는 인접 페이지도 필요하면 따라간다 (1~2 hop까지).
- 모순 블록 (`⚠️ 모순:`)이 보이면 **반드시 답변에 그대로 노출**하고 어느 입장을 따르는지 사용자에게 묻는다.

### 3. 합성 + 출처 표시

- 답변은 마크다운으로. 모든 주장 옆에 `[[페이지명]]` 위키링크로 출처 표시.
- 위키 밖 정보를 끌어다 쓰지 않는다. 위키에 없으면 **"위키에 없음"** 이라고 정직하게 말하고, 사용자가 원하면 `/ingest`로 채우자고 제안.
- 형식은 질문에 맞춤:
  - "이게 뭐야?" → 정의 + 핵심 속성 + 관련 페이지
  - "X와 Y 차이?" → 비교 표
  - "타임라인" → 시간순 리스트
  - "그래프" → mermaid 다이어그램
  - 발표자료 요청 → Marp 슬라이드 마크다운

### 4. 재투입 제안 (Materialize 트리거)

- 답변이 단순 사실 조회 이상의 **합성·결론·비교**를 담고 있으면, 끝에서 묻는다:
  > "이거 `wiki/synthesis/{area}/[주제].md`로 저장할까요?"
- OK 받으면 즉시 페이지로 적재 (CLAUDE.md §3.5, §4.1 frontmatter 준수).
- 적재 후 `wiki/index.md`와 `wiki/log.md`에 한 줄씩 추가:
  ```
  ## [YYYY-MM-DD] query | 짧은제목
  - 합성 페이지로 적재: [[...]]
  - 참조한 페이지: [[...]], [[...]]
  ```

### 5. 휘발 vs 보존 판단

CLAUDE.md §3.5 "무엇을 적재하지 않을 것" 그대로:

- 단순 사실 확인 (예: "n8n이 뭐야?") → 답만 하고 적재 안 함
- 명령 결과 보고 → 적재 안 함
- 사용자가 "기록 남기지 마"라고 한 대화 → 적재 안 함
- **미래의 나/사용자가 다시 찾을 가치가 있는가?** "그렇다"면 적재 제안.

## 가드레일

- 위키에 없는 사실을 추측으로 채우지 않는다. 모르면 "위키에 없음" + ingest 제안.
- WebSearch/WebFetch는 사용자가 명시 요청한 경우에만 사용. 사용했다면 **즉시** `raw/web/YYYYMMDD_[제목].md`에 원본 적재 (CLAUDE.md §3.5 1번).
- 답변에 쓴 모든 주장은 `[[페이지]]` 위키링크로 출처 명시. 출처 없는 단정은 금지.
- 답변에서 조용히 위키 페이지를 *수정*하지 않는다. 수정은 `/ingest` 또는 명시적 적재 제안 후에만.
