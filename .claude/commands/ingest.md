# /ingest — 새 소스를 위키에 통합

CLAUDE.md `§3.1 Ingest` 워크플로우를 그대로 수행한다. 사용자(강동이)는 큐레이터, 너(Claude)는 사서다.

## 입력

- 인자: `$ARGUMENTS`
  - 비어 있으면 → `raw/inbox/`, `Clippings/`, `raw/external/`, `raw/web/` 중 아직 `wiki/sources/`에 대응 페이지가 없는 후보를 나열하고 사용자에게 어느 것부터 처리할지 묻는다.
  - 파일명 또는 키워드가 있으면 → 그걸 우선 매칭해서 처리. 1건만 처리하는 게 기본. 사용자가 "여러 개 한꺼번에"라고 명시한 경우에만 배치.

## 절차 (반드시 이 순서)

### 1. 읽기 + 1차 보고

- 대상 원본을 Read로 끝까지 읽는다 (큰 파일도 끊지 말 것).
- 사용자에게 **핵심 takeaway 3~5개**를 먼저 보고한다.
- 함께 다음을 제안한다:
  - 이 소스가 어떤 **type × area** 페이지들을 만들/갱신할지 (CLAUDE.md §3.4)
  - 새로 생길 엔티티/개념 후보 목록
  - 기존 페이지와 충돌·보강될 만한 지점

> 여기서 사용자가 OK 또는 수정 지시를 줄 때까지 멈춘다. 자동모드여도 분류·생성 범위 합의는 받는다.

### 2. 소스 페이지 작성

OK 받으면 즉시 `wiki/sources/{area}/YYYYMMDD_[제목].md` 생성. 반드시:

- CLAUDE.md §4.1 frontmatter (title, type=source, area, tags, created, updated, sources)
- 핵심 요약 (3~7개 bullet)
- 중요 인용 원문 (블록인용 + 가능하면 위치)
- 출처 메타 (URL, 저자, 발행일, 가져온 날짜, 원본 경로 `raw/...`)

### 3. 엔티티/개념 페이지 갱신

등장한 인물·조직·제품·도구는 `wiki/entities/{area}/`, 개념·프레임워크는 `wiki/concepts/{area}/`.

- **이미 있으면 갱신**: 새 사실은 추가, 기존 주장과 충돌하면 덮어쓰지 말고 `> ⚠️ 모순:` 블록으로 둘 다 보존하고 출처 링크.
- **없으면 생성**: §4.1 frontmatter + 본문 + sources 위키링크.
- 한 소스가 보통 10~15개 페이지를 건드린다. 적게 건드리면 누락이 의심된다.
- 모든 주장 옆에 `[[소스페이지]]` 위키링크로 출처 표시 (Provenance).

### 4. 인덱스 갱신

`wiki/index.md`를 읽고, **area → type** 순서로 새/변경 페이지를 한 줄씩 등재. 이미 있으면 한 줄 설명만 갱신.

### 5. 로그 추가

`wiki/log.md` **맨 아래에 append-only**:

```
## [YYYY-MM-DD] ingest | 짧은제목
- 새 소스 페이지: [[...]]
- 신규 엔티티/개념: [[...]], [[...]]
- 갱신: [[...]]
- 모순 플래그: 0건 또는 [[페이지]]에서 1건
```

날짜는 `currentDate` 값을 사용. prefix 패턴 `## [YYYY-MM-DD] ingest | ...` 엄수 (grep 호환).

### 6. 마무리 보고

사용자에게 다음을 한 메시지로 정리:

- 만든 파일 / 수정한 파일 (경로)
- 생긴 위키링크 그래프 (어떤 엔티티가 어떤 소스/개념에 연결됐는지)
- 다음에 ingest할 만한 후보 (inbox 잔여물)
- 옵시디언 깃이 5분 안에 자동 푸시한다는 안내는 생략 (CLAUDE.md §8에 이미 있음)

## 가드레일

- `raw/`는 **읽기 전용**. 절대 수정·삭제하지 않는다.
- 사용자가 직접 만진 흔적이 있는 wiki 페이지(`updated` 미스매치, 수동 편집 흔적)는 덮어쓰지 말고 먼저 알린다.
- 비밀번호·API키·개인정보가 보이면 적재 거부 + 즉시 사용자에게 알림 (CLAUDE.md §3.5).
- area가 시드 5개에 안 맞으면 새 area를 사용자 승인 받고 만든다 (§3.4 4번).
- 위키링크는 파일명만 적는다 — `[[n8n]]` OK, `[[wiki/entities/자동화/n8n]]` 금지.
