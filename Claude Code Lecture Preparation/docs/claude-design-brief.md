# Claude Design 핸드오프 브리프 (1회용)

> **목적:** 이 프로젝트의 슬라이드 톤앤매너를 Claude Design(claude.ai의 디자인 기능)으로 확정하기 위한 브리프.
> **빈도:** 원칙적으로 1회 (전면 디자인 개편 시에만 재실행).
> **수신물:** Claude Design 산출물(HTML+CSS) → `slide-designer/resources/design-tokens.md`로 토큰화.

---

## 1. 컨텍스트

CCFM AX팀은 Claude / 기타 AI 도구 교육 슬라이드를 **반복 생성**해야 한다. 매번 디자인이 달라지지 않도록 톤앤매너를 먼저 고정하고, 이후 가이드들은 동일한 디자인 시스템을 공유한다.

- 대상: 외부 교육 요청 조직(1순위), CCFM 사내 실무자(2순위)
- 목적: 신뢰감 있고 정돈된 교육 자료, 하지만 딱딱하지 않게
- 레퍼런스 느낌: Anthropic 공식 문서의 여백·타이포 리듬

## 2. 고정 제약 (바꾸지 말 것)

- **비율 16:9**
- **폰트**: Pretendard 우선, Noto Sans KR 폴백 (한글 본문)
- **메인 컬러**: CCFM 브랜드 블루 `#1B4FD8` — 표지 헤더나 주요 강조 요소에 사용
- **단일 HTML + CSS 파일** (외부 CSS 프레임워크, 아이콘 라이브러리, 이미지 CDN 금지)
- **PDF 변환 호환**: `@page` + `@media print` 규칙 필수, `print-color-adjust: exact`
- **컨테이너 고정 높이** (overflow: hidden) — 콘텐츠가 넘치면 내부 요소를 줄이는 구조

## 3. 결정을 맡기는 항목 (Claude Design이 제안)

- **해상도**: 960×540 / 1280×720 / 1920×1080 중 어느 것이 적절한지 제안
- **보조 컬러 팔레트**: 정보(info) / 팁(tip) / 경고(warn) / 성공(success) 4색 — 메인 블루와 조화되는 톤
- **타이포 위계**: 표지 제목 · 슬라이드 헤더 · H2 · H3 · 본문 · 코드 · 캡션의 크기/굵기
- **여백 시스템**: 슬라이드 외곽 패딩, 섹션 간 마진, 그리드 갭
- **그림자·라운드**: 카드/슬라이드 컨테이너의 shadow·border-radius 수치
- **카드·코드 블록 구체 스타일**: 배경·테두리·내부 여백·서체
- **아이콘 전략**: 이모지 유지 vs inline SVG 채택 (파일 안에 포함)

## 4. 샘플로 만들어볼 슬라이드 2장

**디자인 시안을 판단하기 위한 샘플 콘텐츠**일 뿐, 이게 실제 가이드 내용이 되는 건 아님.

### [1장] 표지 슬라이드
- 도구명: `Claude Code`
- 부제: `AI 페어 프로그래밍 CLI`
- 발표 메타: `CCFM AX팀 · 2026.04.22`

### [2장] 3컬럼 기능 카드 슬라이드
- 제목: `Claude Code로 할 수 있는 것`
- 카드 3장:
  1. ① 코드 작성·수정 · 한 줄 설명 · 예시 명령어
  2. ② 파일 탐색·검색 · 한 줄 설명 · 예시 명령어
  3. ③ 명령어 실행 · 한 줄 설명 · 예시 명령어
- 각 카드에 이모지 + 1줄 설명 + 예시 명령어(코드 블록)

## 5. Claude Design에 전달할 프롬프트

아래 전체를 복사해서 claude.ai 의 디자인/아티팩트 기능에 붙여넣기:

```
CCFM AX팀의 AI 교육 슬라이드 톤앤매너를 확정하기 위한 시안을 만들어줘.
이 시안은 한 번 확정 후 여러 교육 가이드에 공통 적용되므로,
"재사용 가능한 디자인 시스템"을 전제로 해야 해.

[고정 제약]
- 비율 16:9, 단일 HTML+CSS 파일
- 폰트: Pretendard(우선), Noto Sans KR(폴백) — 한글 본문
- CCFM 브랜드 메인 블루: #1B4FD8 (표지 헤더 등 주요 강조)
- 외부 CSS 프레임워크·아이콘 라이브러리·이미지 CDN 금지
- 브라우저 Ctrl+P로 PDF 변환 가능하게: @page, @media print, print-color-adjust: exact
- 컨테이너 고정 높이(overflow:hidden) — 콘텐츠가 넘치면 내부 요소 축소

[네가 결정·제안할 것]
- 해상도: 960×540 / 1280×720 / 1920×1080 중 하나 (이유와 함께)
- 보조 컬러 팔레트 4종: 정보(info) / 팁(tip) / 경고(warn) / 성공(success)
- 타이포 위계: 표지 제목 / 헤더 / H2 / H3 / 본문 / 코드 / 캡션
- 여백 시스템: 외곽 패딩, 섹션 간격, 그리드 갭
- 그림자·둥근 모서리 수치
- 카드·코드 블록 스타일
- 아이콘: 이모지 유지 or inline SVG

[톤 레퍼런스]
- Anthropic 공식 문서 느낌의 여백과 타이포 리듬
- 신뢰감 있고 정돈된 기업 교육 자료, 하지만 딱딱하지 않게
- 정보 위계가 한눈에 보이는 차분한 레이아웃

[샘플 콘텐츠 — 시안 판단용]

1장) 표지
  - 도구명: Claude Code
  - 부제: AI 페어 프로그래밍 CLI
  - 메타: CCFM AX팀 · 2026.04.22

2장) 3컬럼 기능 카드
  - 제목: Claude Code로 할 수 있는 것
  - 카드 3장, 각 카드에 [이모지] + [1줄 설명] + [예시 명령어 코드블록]
    1) 코드 작성·수정
    2) 파일 탐색·검색
    3) 명령어 실행

[결과물 형식]
- 가능하면 2가지 디자인 버전(A/B)으로 제안 — 선택권 있게
- CSS는 :root 변수(커스텀 프로퍼티)로 토큰화 — 재사용·수정 용이
- 각 CSS 값 옆에 짧은 의도 주석 ("그림자: 미묘하게", "카드 패딩: 여유 있게" 등)

완성되면 이 artifact 전체를 핸드오프할 예정이야.
```

## 6. 수신 후 절차

Claude Design에서 artifact를 받으면:

1. artifact HTML/CSS 전체를 이 저장소에 붙여넣기 (예: `docs/claude-design-output.html`)
2. CSS `:root` 토큰 값들을 `slide-designer/resources/design-tokens.md` 에 표 형태로 정리
3. `slide-designer/resources/css-template.md` 에 재사용 가능한 CSS 베이스 작성
4. `slide-designer/SKILL.md`의 TBD 섹션을 실제 값 테이블로 교체
5. `SPEC.md` / `CLAUDE.md`의 "⏸ Claude Design 핸드오프 대기" 섹션을 확정 내용으로 업데이트
6. `dev/active/mvp-setup/mvp-setup-state.md`에 "디자인 확정 (YYYY-MM-DD)" 섹션 추가
