# 콘텐츠 브리프 템플릿 (매 가이드마다 사용)

> **목적:** 새 교육 가이드를 만들 때마다 guide-generator 스킬이 **레퍼런스 기반으로 정확한 콘텐츠를 생성**하도록 하는 입력 브리프.
> **빈도:** 매 가이드 1회.
> **사용법:** 아래 템플릿을 복사해 채운 뒤 Claude Code에게 전달. 빈 칸이 있으면 guide-generator가 되물을 것.

---

## 템플릿 (복사해서 쓰세요)

```yaml
# === 콘텐츠 브리프 ===

조직명: [예: CCFM / 외부 조직명]
주제: [예: Claude Design 기능 소개 / Artifacts 실습 / Claude Code 설치]
가이드 유형: [install | feature-intro | hands-on | troubleshooting]
발표일: [YYYY-MM-DD]
발표자: [선택]

대상 청중:
  - 수준: [비개발자 / 주니어 개발자 / 시니어 / 혼합]
  - 사전 지식: [예: Claude 계정은 있음, CLI 경험 없음]

학습 목표 (3~5개):
  - [ ]
  - [ ]
  - [ ]

레퍼런스 (최소 1개 이상 필수):
  - URL: [예: https://www.anthropic.com/news/claude-design]
  - URL: [예: https://docs.anthropic.com/...]
  - YouTube: [링크 + 요약/스크립트 첨부]
  - 문서: [파일 경로 또는 붙여넣은 요약 텍스트]

강조 포인트 (선택):
  - [예: "협업 디자인 워크플로우" 부분을 핵심으로 다뤄줘]

제외·주의 (선택):
  - [예: 가격 정책은 자주 바뀌므로 슬라이드에 넣지 말 것]

예시·실습 선호:
  - [실제 사용 시나리오 1~2개 서술]

# === 끝 ===
```

## 각 필드 의미

### `조직명` · `발표자` · `발표일`
표지 슬라이드 메타데이터로 치환. 폴더명(`outputs/[조직명]_[주제]_YYYYMMDD`)에도 사용.

### `주제`
슬라이드 제목과 파일명에 반영. 너무 길면 축약됨.

### `가이드 유형`
템플릿 선택을 결정. 모호하면 `feature-intro` 기본.

| 유형 | 언제 |
|------|------|
| `install` | 설치·셋업·환경 구성 가이드 |
| `feature-intro` | 새 기능·도구 소개 (가장 일반적) |
| `hands-on` | 실습 중심 — 따라하며 배우는 구성 |
| `troubleshooting` | 자주 발생하는 문제 해결 |

### `대상 청중`
난이도 조정에 사용. 비개발자 대상이면 용어 풀어 설명, 시니어면 압축.

### `학습 목표`
가이드가 끝나면 청중이 뭘 할 수 있어야 하는가. 이게 곧 마지막 장의 "정리" 섹션이 됨.

### `레퍼런스` (★ 필수)
**없으면 guide-generator는 콘텐츠 생성을 거부한다.** 할루시네이션 방지 원칙.

- URL: `WebFetch`로 원문 가져와 파싱
- YouTube: 요약이나 스크립트 같이 첨부 (링크만 있으면 불충분)
- 문서: 파일로 저장 후 경로 제공, 또는 요약 붙여넣기

여러 개 주면 교차 참조로 정확도 상승.

### `강조 포인트` · `제외·주의`
레퍼런스에 있는 내용 중 우선순위 조정. 시간 제약(30분 강의 등)이나 청중 관심사에 맞춤.

### `예시·실습 선호`
실제 사용 시나리오. 추상적 설명보다 "이런 경우에 이렇게 쓴다" 가 머리에 남음.

## 흐름 예시

```
사용자: "Claude Design 기능 소개 가이드 만들어줘"

guide-generator: "콘텐츠 브리프를 먼저 받을게요. 아래 템플릿을 채워주세요.
                  특히 레퍼런스는 필수입니다 (공식 문서 URL, 블로그 포스트, 영상 요약 등)."
       ↓
사용자: [브리프 작성해서 전달]
       ↓
guide-generator:
  1. 레퍼런스 수집 (WebFetch https://www.anthropic.com/news/claude-design ...)
  2. 핵심 개념 추출
  3. 6장 슬라이드 구조 매핑
  4. slide-designer 호출 → HTML 렌더링
  5. outputs/CCFM_ClaudeDesign_20260422/index.html 생성
  6. 출처 목록 마지막 장에 명시
       ↓
사용자: 브라우저로 열고 Ctrl+P → PDF
```

---

## 작성 시 주의

- 레퍼런스는 **원문의 URL이나 본문**이 가장 정확. "대충 이런 기능이야" 식의 1줄 설명은 생성 품질이 떨어짐
- YouTube 링크만 주는 것은 부족함. 영상 요약·스크립트를 함께 제공
- 학습 목표가 너무 많으면(7개 이상) 슬라이드가 과적재됨. 3~5개로 압축
- 같은 주제 재생성 시에는 이전 브리프를 복사해 수정하면 빠름
