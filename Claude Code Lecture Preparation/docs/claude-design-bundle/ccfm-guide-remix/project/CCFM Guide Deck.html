01 · What & Why

Claude Code란 무엇인가?

터미널에서 직접 돌아가는 AI 코딩 에이전트 — 채팅창이 아니라 당신의 작업 환경 안으로 들어옵니다.

01

터미널 네이티브 (Terminal-native)

브라우저·IDE 플러그인이 아닌 CLI에서 실행. 당신의 파일, 커밋, 환경변수, 툴체인에 직접 접근합니다.

02

맥락 이해 (Context-aware)

현재 디렉토리의 전체 코드베이스를 읽고 수정합니다. 파일 하나씩 복붙할 필요가 없어요.

03

에이전틱 실행 (Agentic)

명령 대신 목표를 줍니다. Claude가 계획을 세우고, 파일을 고치고, 테스트까지 돌려봅니다.

04 · Installation · Windows

Windows 설치 — 네 단계

Windows는 WSL2(Ubuntu) 위에서 실행하는 것이 공식 권장 방식입니다.

Step 01

WSL2 설치

PowerShell을 관리자 권한으로 열고 실행. 재부팅 필요.

wsl --install
Step 02

Ubuntu 계정 생성

재부팅 후 터미널에 Ubuntu 창이 뜹니다. 사용자명 · 비밀번호 설정.

username / password
Step 03

Node.js 설치

WSL(Ubuntu) 터미널에서 NodeSource LTS를 설치합니다.

sudo apt install -y nodejs npm
Step 04

Claude Code 설치

npm 글로벌 설치 후 첫 실행 시 로그인 안내가 뜹니다.

npm install -g @anthropic-ai/claude-code

06 · Trade-offs

장단점 — 솔직하게

모든 업무가 Claude Code에 적합한 건 아닙니다. 맞는 일에 쓰세요.

✓  Pros · 장점

  • 로컬 파일/커맨드 직접 접근 — 복붙 없음
  • 반복 작업(데이터 정제, 리팩토링) 압도적 속도
  • 프로젝트 맥락 기반 응답 — 품질 일관성
  • 비개발자도 자연어로 스크립트 생성 가능
  • Git, 테스트, 터미널 통합 — 실무 적합도 높음

✗  Cons · 단점

  • WSL 등 초기 환경 세팅 허들 (Windows)
  • 대형 코드베이스에서 토큰 비용 증가
  • 자동 실행 권한 — 승인 전 반드시 검토 필요
  • 실시간 UI/디자인 피드백에는 부적합
  • 사내 데이터 반출 관련 내부 정책 확인 필수

주의사항
Do

작업 전 Git 커밋을 습관화하세요. 되돌릴 지점이 반드시 필요합니다.

Do

민감 데이터가 포함된 파일은 로컬 샘플본으로 먼저 테스트하세요.

Don't

자동 실행(--dangerously-skip-permissions) 플래그는 검증된 스크립트에만.

Don't

고객사 원본 데이터·자격증명을 그대로 컨텍스트에 넣지 마세요. 사내 정책을 먼저 확인.

템플릿 사용 가이드

콘텐츠를 바꿔도 레이아웃이 깨지지 않는 네 가지 원칙.

1 슬롯(data-slot) 표기를 따르세요

각 콘텐츠 영역에는 data-slot="title · max 2 lines" 같은 제약이 붙어 있습니다. Tweaks에서 Show slots를 켜면 슬롯이 시각화됩니다.

2 텍스트 길이 한도를 지키세요

  • Title: 최대 2줄 (약 30자)
  • Lede: 최대 3줄 (약 80자)
  • 3-up 카드 설명문: 최대 3줄
  • Do/Don't 각 항목: 1문장 원칙

3 레이아웃을 새로 만들지 마세요

슬라이드 유형은 Cover · TOC · 3-up · Before/After · Steps · Code · Pros/Cons · UseCase · Divider로 고정. 새 주제는 기존 유형 중에서 고르세요.

4 Tweaks로 톤을 조정하세요

  • 악센트 색 — 주제별로 구분 (Claude=Rausch)
  • Dark/Light — 장표 성격에 따라
  • Font scale — 현장 화면 크기 맞춤
  • Footer on/off — 인쇄 vs. 스크린
