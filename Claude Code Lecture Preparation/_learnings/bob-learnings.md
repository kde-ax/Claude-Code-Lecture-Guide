## 2026-04-23 — 슬라이드 구조 교훈 (v2 → v3 반복 과정에서 도출)

### [미세조정] 푸터 Zone 확보 필수
- 슬라이드 720px 중 하단 62px은 절대 위치 푸터(페이지 번호) 영역
- 본문 총 높이가 608px을 넘으면 푸터와 겹쳐 "본문이 가려지는" 증상
- 반영: `slide-designer/SKILL.md` "구조적 함정" 섹션에 규칙 명시
- 반영: `design-tokens.md` Spacing 섹션에 Footer Zone 서브섹션 추가
- 반영: `guide-generator/SKILL.md` 생성 프로세스 Step 7 (Overflow 체크) 추가

### [미세조정] Cover Graphic 기본 높이
- 520px → **440px** 축소
- 전체 720 − header 40 − padding 112 − footer 60 = ~508px 상한
- 440px가 안전·시각적으로도 균형
- 반영: `template/deck.html` · `template/deck-print.html` 기본값 업데이트

### [프로세스] 구조 이슈는 SKILL에 박아야 재발 안 함
- 단일 HTML만 고치면 다음 가이드에서 또 같은 실수 반복
- 사용자 지적: "md파일이나 스펙 자체를 변경시켜서 그 다음에도 같은 문제가 안생기게"
- 교훈: 콘텐츠 버그 ≠ 구조 버그. 구조 버그는 슬라이드 규격 레벨에서 고정해야 함

### [프로세스] 버전 규칙 실전 적용 효과
- `index_v1.html` → `v2` → `v3` 로 분기 저장이 잘 작동
- 사용자가 "v2 에서 어떻게 바뀌었는지 확인"·"v1로 롤백" 용이
- 각 버전의 의도가 README 이력 표에 기록되어 추적 가능
